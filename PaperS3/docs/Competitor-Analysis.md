## 详细差距分析

### 1. EPUB 支持（最大短板）

**为什么重要：**
- 市面上 90% 电子书是 EPUB 格式
- EPUB 自带目录、封面、排版信息
- 用户不需要自己转 TXT

**实现方案：**
- 集成 **miniz** (ZIP 解压，ESP32 ROM 内置)
- 集成 **TinyXML2** 解析 OPF + XHTML
- 解析 `META-INF/container.xml` → `OEBPS/content.opf`
- 按 spine 顺序渲染章节

**工作量：** 大（2-3 天）

### 2. 最近阅读列表

**为什么重要：**
- 每天打开阅读器，90% 是继续读上次的书
- 省去文件浏览器翻找的步骤

**实现方案：**
- 保存最近 10 本书到 `/progress/recent.json`
- 书架首页显示「继续阅读」+ 最近列表
- 显示每本书的阅读进度百分比

**工作量：** 小（2-3 小时）

### 3. 快速页码跳转

**为什么重要：**
- 1000+ 页的书，从 1% 跳到 50% 要翻 500 次
- 用户知道章节名但不知道页码

**实现方案：**
- 菜单新增「跳转页码」
- 数字输入界面（类似计算器）
- 或百分比拖动条

**工作量：** 中（半天）

### 4. 书末处理

**为什么重要：**
- 读完一本书应该明确知道「结束了」
- 提供返回书架或打开下一本书的选项

**实现方案：**
- 读到最后一页显示「本书已读完」
- 提供按钮：返回书架 / 打开下一本

**工作量：** 小（1-2 小时）

### 5. 封面显示

**为什么重要：**
- 书架列表显示封面更直观
- 提升视觉体验

**实现方案：**
- TXT 书籍：用文字生成简单封面（书名+作者）
- EPUB 书籍：提取 cover.jpg
- 缓存缩略图到 SD 卡

**工作量：** 中（半天）

### 6. 休眠屏

**为什么重要：**
- 休眠时显示时间/封面/阅读进度
- 墨水屏断电保持画面，适合做时钟

**实现方案：**
- 休眠前渲染自定义画面（时间 + 封面 + 进度条）
- 存储到 SD 卡，唤醒后恢复

**工作量：** 中（半天）

### 7. OTA 更新

**为什么重要：**
- 不用拔 SD 卡、不用连电脑就能升级
- 迭代开发更方便

**实现方案：**
- ESP32 内置 OTA 支持
- WiFi 下载固件到 partition
- 重启后自动切换

**工作量：** 中（半天）

## 迭代计划建议

### 第一阶段：基础体验完善（1-2 天）

1. **最近阅读列表** - 高频使用，工作量小
2. **书末处理** - 完善阅读闭环
3. **快速页码跳转** - 长书必备
4. **封面显示** - 书架美观

### 第二阶段：核心功能突破（2-3 天）

5. **EPUB 支持** - 最大短板，工作量最大
6. **休眠屏** - 提升质感
7. **OTA 更新** - 方便迭代

### 第三阶段：高级功能（可选）

8. **KOReader Sync** - 跨设备同步
9. **OPDS 书库** - 在线书库
10. **屏幕旋转** - 竖屏阅读

## 技术方案参考

### EPUB 解析架构（参考 diy-epub-reader）

```
EPUB File (.epub)
  → miniz (ZIP 解压)
    → container.xml (找到 content.opf)
      → content.opf (清单 + spine)
        → chapter1.xhtml
        → chapter2.xhtml
          → TinyXML2 解析 HTML
            → 文本块列表
              → 分页引擎
                → 屏幕渲染
```

### CrossPoint 的缓存策略

```
.crosspoint/
├── epub_<hash>/
│   ├── progress.bin      # 阅读进度
│   ├── cover.bmp         # 封面
│   ├── book.bin          # 书籍元数据
│   └── sections/
│       ├── 0.bin         # 预渲染章节
│       ├── 1.bin
│       └── ...
```

## 结论

我们的固件在 **中文 TXT 阅读体验** 上已经做得很好（灰度字库、排版、编码检测、章节识别）。

但最大的 **结构性短板** 是：
1. **不支持 EPUB** - 限制了书籍来源
2. **没有最近阅读** - 每天多几步操作
3. **缺少页码跳转** - 长书体验差

建议优先补齐这 3 项，然后再考虑 EPUB 这个大工程。
